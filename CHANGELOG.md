  可随时打开今天的笔记、不想被记录可暂停、静音时保持安静不乱编」等使用说明——用户念一遍即学会怎么用。

## v0.6.1 — 2026-06-19

- 朗读稿(用户念的内容)改为**第一人称的产品/隐私说明**：「这款应用全程在我这台电脑上运行，不联网、
  不上传…录到的声音转成文字后立刻丢弃…」。念一遍既采了声纹，又把隐私承诺自己说了一遍(读比看记得牢)。

## v0.6.0 — 2026-06-19
声纹注册改为「须知 → 开始 → 采集」两阶段引导。
- **第一屏「须知页」**：弹窗即显示但不录音——顶部本地/隐私须知(全程本地、不联网、不上传、不读取其他数据、
  音频即用即弃) + 用途说明 + 一句广告词；中间「开始」按钮。可在 config 的 `enroll_intro` 自定义。
- **点「开始」后**才切到朗读句子 + 进度条并正式录音(解决「弹窗即录、用户没反应过来」)。
- **经多 agent 对抗审查加固**(本次随功能一并修)：
  - 激活策略改为 `ui_common` 引用计数 push/pop——多窗口不再各自盲切 Regular/Accessory 互踩，
    构造失败/异常路径也不会卡住 Dock 图标(push 作为构造最后一步 + close 幂等)。
  - 取消采集立即停旁路(`_enroll=None`)，实时转写当即恢复，不再卡最多 ENROLL_MAX_SEC；
    `_callback` 快照引用，消除与 worker 置 None 的 `None.append` 竞争。
  - 注册会话属性初始化、`_enroll_start` 先停旧定时器、自动收尾「先停表断引用再关窗」、
    `on_start` 失败兜底收尾、注册期间禁开关键词窗、取消键加 Esc。

## v0.5.1 — 2026-06-19
修复：关键词窗口能显示、有光标，但敲键盘没反应、无法编辑。
- 根因：菜单栏 App 是 Accessory(无 Dock)策略，此状态下窗口能显示也能放光标，但**物理按键被系统
  送给真正的前台 App**，不是我们——单靠 activateIgnoringOtherApps 不够。
- 修法：打开窗口时临时把 App 切到 Regular(前台 App)夺回键盘焦点，关窗后再切回 Accessory(恢复无 Dock
  的菜单栏状态)。这是菜单栏 App 弹出可输入窗口的标准做法。

## v0.5.0 — 2026-06-19
关键词窗口可编辑(修 bug) + 新增「识别词库」写法。
- **修复：关键词窗口现在能编辑**。原因：rumps 是无 Dock 的 LSUIElement 进程，非模态窗口拿不到键盘
  焦点→只能看不能打字。改为**模态窗口**(runModalForWindow + makeFirstResponder)，键盘直达，可正常增删改。
- **两种写法，一个文本框**：
  - `错词 = 正词` → 精确纠错(replace)：转写出现错词就替换。
  - 只写 `目标词` → 识别词库(terms)：注入 whisper 的 initial_prompt，从源头让它更可能直接听对该词。
- 为什么不做「相似就改成目标词」的模糊替换：对永久日志而言，模糊匹配会**悄悄改错正确的词**
  (Cloud→Claude 误伤、同音字)，是最糟的失败。识别词库在源头偏置、零误伤，是更安全的等价方案。
- 菜单项「关键词纠错…」→「关键词管理…」；config 新增 `terms` 列表；写回 config 的 replace/terms
  两块保证幂等、保留其余配置。

## v0.4.1 — 2026-06-19
注册朗读稿改为「平常会说的话」。
- 去掉 5 首唐诗 + 字母/数字罗列：ECAPA 声纹是 text-independent，只认音色不认内容，刻意覆盖
  数字/字母/单词是过度工程；真正决定质量的是「足够的自然语音时长」(已由质量驱动保证) 与「平常的
  说话方式」。
- 换成几句日常口语(含一句带数字的童谣，顺带覆盖数字但不刻意)，读着不累、像聊天，避免「朗诵腔」
  与日常音色产生偏差。窗口提示也改成「采的是音色不是内容，读错没关系」。

## v0.4.0 — 2026-06-19
朗读稿改「人人能读」+ 新增关键词纠错管理窗口。
- **注册朗读稿换成小学生都会读的内容**：唐诗 + 数数(一到十) + 字母(ABCDEFG) + 最简单的中英问候语
  (你好/谢谢/再见/Hello/Thank you)。去掉了 Claude/MCP/pangram 等生僻词——声纹只认音色不认内容，
  舒服好读最重要。
- **关键词纠错管理窗口**（`replace_ui.py`）：菜单栏「关键词纠错…」打开。一个可编辑文本框承载全部规则
  (每行 `错误 = 正确`)，增 / 删 / 改 / 批量粘贴都在这一个面里编辑，「保存」即时生效并写回 config.yaml
  的 replace 块(保留其余配置与注释)。不再需要手改 yaml。
- 抽出 `ui_common.py`(按钮回调桥 BtnTarget + 标签 make_label)，供 enroll_ui / replace_ui 共用，消除重复。

## v0.3.0 — 2026-06-19
声纹注册体验重做：自建原生窗口 + 质量驱动采集。
- **自建 Cocoa 注册窗口**（`enroll_ui.py`）取代 TextEdit：统一大字排版、不再因机器/版本字体不一而难读；
  含**实时进度条** + 状态行（已采集有效语音 X/Y 秒、用时、百分比）+ 取消/关闭按钮。
- **质量驱动采集**（替代固定 25s 盲采）：累计「有效语音」秒数（响度过门的帧才算），采够
  `enroll_voiced_sec`(默认 20s) 才主动停止。用户中途发呆只采到静音 → 不会误判完成、不会覆盖旧档；
  `enroll_max_sec`(默认 120s) 硬上限兜底。
- **提取质量**指标：注册后给出各窗嵌入对质心的平均余弦（0~100%），偏低提示在安静环境重采。
- **中英混合朗读稿**：内置稿改为「中文 + 真实术语(Claude/MCP/RAG…) + 英文 pangram + 唐诗」，
  贴合机主中英混说的真实音色，质心更具代表性。可用 `enroll_script` 自定义。

## v0.2.1 — 2026-06-19
声纹注册体验：用户不用再发愁「该说什么」。
- 点「注册我的声音」会弹出一段**朗读稿当提词器**（TextEdit 打开，录音 25s 期间一直可见）。
- 内置默认稿为小学课本级唐诗（静夜思/春晓/登鹳雀楼/悯农/咏鹅）——人人会念、零争议、音素覆盖广，
  比临场瞎说更稳。可在 `config.yaml` 的 `enroll_script` 自定义。

## v0.2.0 — 2026-06-19
治「没说话却冒慢慢慢」「外放视频被记成你」+ 声纹识别。
- **复读幻觉过滤**：whisper 在噪声上会锁进单字/整句复读循环（「慢慢慢…」），固定黑名单结构上抓不住。
  新增结构性检测 `_looped`（8+ 连续同字 / 字符多样性崩塌 / zlib 压缩比），且保留「慢慢慢慢慢」这类正常强调。
- **VAD 加严**：显式传 `vad_threshold`（默认 0.6，原默认 0.5 对贴身麦太松），噪声尖峰不再误开句。
- **转写前三道门**（噪声/外放压根不进 whisper，省算力 + 不污染日志）：
  ① 时长门 `min_speech_ms`（补 VADIterator 缺失的最短句长）② 能量/近场门 `min_rms_dbfs`（挡远场外放视频）
  ③ 声纹门（ECAPA-TDNN 说话人验证，挡近场同语种视频/他人）
- **声纹注册**：菜单栏「注册我的声音」念 ~25s → 「声纹门」开关；菜单显示「上句相似度」便于校准阈值。
  fail-open：未注册/模型缺失一律放行，绝不因声纹故障吞掉机主的话。
- 菜单栏显示「滤除 N」计数；新增 `speaker.py` 声纹子模块；新增依赖 `speechbrain`。
